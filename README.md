# MedBook: Σύστημα Διαχείρισης Ιατρικών Ραντεβού

Το **MedBook** είναι μια ολοκληρωμένη και σύγχρονη διαδικτυακή πλατφόρμα (Web Application) σχεδιασμένη για την αυτοματοποίηση της διαδικασίας κλεισίματος και διαχείρισης ιατρικών ραντεβού, με ενσωματωμένο **AI Chatbot** για άμεση εξυπηρέτηση των ασθενών. 

---

## Βασικά Χαρακτηριστικά (Features)

### Για τον Ασθενή:
- **Χωρίς Εγγραφή:** Γρήγορο κλείσιμο ραντεβού χωρίς ανάγκη δημιουργίας λογαριασμού, μέσω ταυτοποίησης με ΑΜΚΑ.
- **Αποτροπή Double-Booking:** Έλεγχος διαθεσιμότητας σε πραγματικό χρόνο (ραντεβού ανά 30 λεπτά).
- **Έξυπνος Βοηθός (AI Chatbot):** Ενσωμάτωση του Google Gemini API για:
  - Άμεσες ιατρικές συμβουλές βάσει συμπτωμάτων (συνοδεύονται πάντα από αυτόματο ιατρικό disclaimer/άρνηση ευθύνης).
  - Δυνατότητα τροποποίησης (αλλαγή ώρας/ημέρας) ή ακύρωσης ραντεβού μέσα από το περιβάλλον του chat, μέσω ενός state machine.

### Για τον Ιατρό (Dashboard):
- **Ασφαλής Σύνδεση:** Πρόσβαση στο διαχειριστικό περιβάλλον με κρυπτογραφημένο κωδικό πρόσβασης.
- **Στατιστικά:** Άμεση προβολή των συνολικών, εκκρεμών, επιβεβαιωμένων και ακυρωμένων ραντεβού.
- **Πλήρης Έλεγχος:** Δυνατότητα αναζήτησης, φιλτραρίσματος, επιβεβαίωσης, επεξεργασίας στοιχείων και διαγραφής ασθενών.

## Τεχνολογίες & Αρχιτεκτονική

Το σύστημα βασίζεται σε αρχιτεκτονική Client-Server:
- **Frontend:** HTML5, CSS3 (Custom styling, Responsive Design), Vanilla JavaScript.
- **Backend:** PHP 8 (Session management, State machine για το Chatbot).
- **Βάση Δεδομένων:** MySQL (Relational DB).
- **Τεχνητή Νοημοσύνη:** Google Gemini API (gemini-flash-latest).
- **Local Server:** MAMP (για macOS).

### Ασφάλεια & GDPR
- **PDO Prepared Statements:** Αποτροπή επιθέσεων SQL Injection σε κάθε query προς τη βάση.
- **CSRF Tokens:** Προσθήκη "κρυφών" tokens σε κάθε φόρμα και POST αίτημα ενάντια σε πλαστογραφήσεις (Cross-Site Request Forgery).
- **Bcrypt Hashing:** Ασφαλής κρυπτογράφηση των κωδικών πρόσβασης του ιατρού.
- **Data Validation:** Έλεγχος εγκυρότητας ΑΜΚΑ (11 ψηφία) και sanitization των εισροών του χρήστη.

## Οδηγίες Εγκατάστασης (Installation & Setup)

Ακολουθήστε τα παρακάτω βήματα για να τρέξετε το project τοπικά στον υπολογιστή σας:

### 1. Προαπαιτούμενα
- Εγκατάσταση του **MAMP** (για macOS) ή XAMPP (για Windows/Linux).
- Βεβαιωθείτε ότι η έκδοση της PHP είναι **8.0+**.
- Ενεργοποιημένο το `cURL` extension της PHP (για την επικοινωνία με το Gemini API).

### 2. Μεταφορά Αρχείων
Τοποθετήστε ολόκληρο τον φάκελο του project (`medical_appointments`) μέσα στον φάκελο `htdocs` του MAMP.
Η διαδρομή στο macOS πρέπει να είναι: `/Applications/MAMP/htdocs/medical_appointments/`

### 3. Ρύθμιση Βάσης Δεδομένων
1. Ανοίξτε τον MAMP και ξεκινήστε τους Servers (Apache & MySQL).
2. Ανοίξτε το **phpMyAdmin** (συνήθως στο `http://localhost:8888/phpMyAdmin`).
3. Δημιουργήστε μια νέα βάση δεδομένων με το όνομα **`medical_db`** (Collation: `utf8mb4_unicode_ci`).
4. Κάντε **Import** το αρχείο `/sql/medical_db.sql` που βρίσκεται μέσα στον φάκελο του project. Αυτό θα δημιουργήσει τους απαραίτητους πίνακες (`doctors`, `appointments`) και θα προσθέσει τα δοκιμαστικά δεδομένα.

### 4. Σύνδεση στο Dashboard (Login)
Για να συνδεθείτε στο Διαχειριστικό ως Ιατρός, πηγαίνετε στη σελίδα `/login.php` και χρησιμοποιήστε τα εξής στοιχεία:
- **Username:** `admin`
- **Password:** `doctor123`

## Ρύθμιση AI Chatbot (Gemini API)
Για να λειτουργήσει ο ψηφιακός βοηθός, απαιτείται το API Key από τη Google.
1. Μεταβείτε στο [Google AI Studio](https://aistudio.google.com/) και δημιουργήστε ένα νέο API Key.
2. Ανοίξτε το αρχείο `chatbot_api.php`.
3. Βρείτε τη μεταβλητή με το API Key και επικολλήστε το δικό σας κλειδί.

## Βασική Δομή Αρχείων (Project Structure)
- `index.php`: Αρχική σελίδα (Hero section, περιήγηση).
- `appointment.php`: Φόρμα κλεισίματος ραντεβού από τον ασθενή.
- `dashboard.php`: Πίνακας ελέγχου και διαχείρισης για τον ιατρό.
- `login.php`: Σελίδα αυθεντικοποίησης (authentication).
- `chatbot_api.php`: Το backend module επικοινωνίας με το Gemini API & το State Machine διαχείρισης ραντεβού.
- `config/db.php`: Αρχείο διασύνδεσης (PDO) με τη βάση δεδομένων.
- `css/style.css`: Βασικό αρχείο μορφοποίησης της εφαρμογής.
- `js/chatbot.js`: Frontend κώδικας για την εμφάνιση και λειτουργία του παραθύρου Chatbot.
- `sql/medical_db.sql`: Το dump της βάσης δεδομένων για το installation.
